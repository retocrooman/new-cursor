# UI

- 画面構成
  - 左：タスクリストを置く
  - 中央：タスク詳細・ダッシュボードを置く
  - 右：司令官とのチャットを置く
- エージェント設定
  - `/agents` でワーカー用エージェントのイベント購読（eventTypes）を設定する
- タスクリスト
  - レポジトリ名を必ず表示する
  - 実行時間を表示する
  - 消費トークン数を表示する
  - 最後のイベントを表示する
  - 状態をアイコンで表示する
- ダッシュボード
  - 重要な意思決定リストを表示する
  - イベントヒストリーを表示する
  - 作業ファイルの一覧と要約を表示する
  - コードの変更量を表示する
  - ブランチ名を表示する
  - レポジトリ名を表示する
- 司令官チャット
  - 起票時は、タスクタイトルを必ずユーザーに確認する
  - 起票時は、ベースブランチを必ずユーザーに確認する
  - 起票時は、ブランチ名を必ずユーザーに確認する
  - 起票時は、背景を必ずユーザーに確認する
  - 起票時は、検証項目を必ずユーザーに確認する
- イベントフロー
  - 目的：任意のイベント連鎖（いつ・何が起きたら → 何をする → 次に何を送る）を定義する汎用オートメーション画面。タスク工程テンプレートに限定しない。
  - 遷移：3カラムの運用UIとは別画面。ナビゲーションから開く。
  - トリガー：イベント種別と条件（発火条件）を指定する。
  - アクション：実行内容（通知・API 呼び出し・状態更新など）を指定する。
  - 発火：アクション完了後に送る次イベントを指定する。
  - 保存：定義は司令室（API+DB）に保存し、イベント駆動のルーティングに使う。
  - 連携：トリガー条件を満たしたとき、司令室が SQS 経由で Worker に作業依頼を届ける。
